## Phase 1: 基盤ツールのインストール

### 手順1-1: Docker Desktopのインストール

**📍 実行ディレクトリ**: 任意（ダウンロードフォルダ等）

| 4. 再起動 | Windows再起動 | Docker Desktopの有効化に必要 | - |
| 5. 起動確認 | Docker Desktopを起動 | 正常にインストールされたか確認 | - |
| 6. バージョン確認 | `docker --version` | Dockerが使用可能か確認 | - |

**期待される出力**:
```
Docker version 24.0.7, build afdd53b
```

**トラブルシューティング**:
- WSL 2エラー → `wsl --install` を管理者権限のPowerShellで実行

## トラブルシューティング早見表

| エラー | 原因 | 解決策 | 実行ディレクトリ |
|-------|------|-------|----------------|
| `docker: command not found` | Docker未インストール | Docker Desktop インストール | - |
| `Cannot connect to the Docker daemon` | Docker Desktop未起動 | Docker Desktopを起動 | - |
| `Port 5432 is already in use` | PostgreSQL既に起動中 | `docker compose down` → 再起動 | プロジェクトルート |
| `tsx is not recognized` | tsxグローバル未インストール | `npm install -g tsx` | 任意 |
| `vite is not recognized` | viteグローバル未インストール | `npm install -g vite` | 任意 |
| `prisma command not found` | Prisma未インストール | `npm install` 再実行 | server/ |
| `Error: connect ECONNREFUSED` | Dockerコンテナ未起動 | `docker compose up -d` | プロジェクトルート |
