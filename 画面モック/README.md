## デザインの特徴

### 顧客向け画面
- **スマホ専用デザイン**：375px幅のフレーム
- **タップしやすいボタン**：最小44x44px
- **カラーパレット**：
  - プライマリ: #4CAF50（緑）
  - セカンダリ: #FF9800（オレンジ）
  - アクセント: #2196F3（青）
  - エラー: #F44336（赤）
  - 背景: #F5F5F5（ライトグレー）
- **アニメーション**：注文完了時のスケールインアニメーション
- **ステータスバッジ**：色分けで視覚的に状態を表示

### 店舗管理画面
- **PC/タブレット対応**：レスポンシブデザイン
- **テーブル形式**：データ一覧表示
- **ステータスバッジ**：視覚的な状態表示
- **カードレイアウト**：ダッシュボードのサマリー表示
- **トグルスイッチ**：売り切れ管理
- **モーダル表示**：QRコード表示

## 画面フロー

### 注文フロー（顧客）
1. C-S01（セッション開始）→ 来店人数入力
2. C-S02（メニュー一覧）→ 商品を閲覧
3. C-S03（商品詳細）→ 数量選択・カート追加
4. C-S04（カート）→ 注文確定
5. C-S05（注文完了）→ メニューに戻る

### その他の機能（顧客）
- C-S06（注文履歴）：注文状況を確認
- C-S07（店員呼び出し）：スタッフに連絡
- C-S08（会計）：会計依頼・キャンセル
- C-S09（検索）：商品名で検索

### 管理フロー（店舗）
1. A-S01（ログイン）→ 認証
2. A-S02（ダッシュボード）→ 全体状況確認
3. A-S03（注文ダッシュボード）→ 注文対応
4. A-S11（会計処理）→ 会計完了
5. A-S12（売上管理）→ レポート確認

## 備考

- 画像は絵文字で代用しています
- 実装時にはフォント（Noto Sans JP）の読み込みが必要です
- JavaScriptによる動的な動作は含まれていません
- すべての画面は静的なHTMLモックです

## 設計ガイドライン準拠

✅ **コントラスト比**: 4.5:1以上（AA準拠）  
✅ **タップターゲット**: 最小44x44px  
✅ **フォントサイズ**: 本文14px〜16px  
✅ **角丸**: ボタン8px、カード12px  
✅ **レスポンシブ**: スマホ専用（320px〜480px）、PC/タブレット対応
